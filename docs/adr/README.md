# Architecture Decision Records

Use ADRs for decisions that future agents must understand before changing architecture, database contracts, permissions, release flow, or cross-module behavior.

## Naming

```text
ADR-0001-short-title.md
```

## Minimum Content

- Status
- Date
- Context
- Decision drivers
- Options considered
- Decision
- Consequences
- Risks and mitigations
- Validation plan
- Revisit conditions
